# OcularAI — GitHub Pages Deployment Guide

## Step 1 — Convert the Model (one-time)

Run this from the `eye_classification` folder:

```bash
py convert_model.py
```

This creates `model/model.json` + `model/group1-shard*.bin` files.
The script applies **uint8 quantization**, keeping each shard under 20 MB.

---

## Step 2 — Initialize a Git Repository

```bash
cd eye_classification
git init
git add .
git commit -m "Initial commit: OcularAI eye disease classifier"
```

---

## Step 3 — Create a GitHub Repository

1. Go to https://github.com/new
2. Create a **public** repository (e.g. `ocular-ai`)
3. Do **NOT** initialize with README (you already have files)

---

## Step 4 — Push to GitHub

```bash
git remote add origin https://github.com/YOUR_USERNAME/ocular-ai.git
git branch -M main
git push -u origin main
```

---

## Step 5 — Enable GitHub Pages

1. Open your repo on GitHub
2. Go to **Settings** → **Pages** (left sidebar)
3. Under **Source**, select:
   - Branch: `main`
   - Folder: `/ (root)`
4. Click **Save**

Your site will be live in ~1 minute at:
```
https://YOUR_USERNAME.github.io/ocular-ai/
```

---

## File Size Notes

- GitHub has a **100 MB per-file limit**
- The model shards are set to 20 MB each — well within limits
- Total model size after quantization: ~60–80 MB across multiple shards

If any shard exceeds 100 MB, edit `convert_model.py` and reduce:
```python
weight_shard_size_bytes=10 * 1024 * 1024  # 10 MB instead of 20 MB
```
